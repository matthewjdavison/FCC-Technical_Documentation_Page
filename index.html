HTML Markup

HTML stands for 'Hyper Text Markup Language'. It uses 'Tags' to define what type of information is being shown on the page.

A HTML page usually includes a basic frame of a 'Head' which includes the meta data and the 'Body' which is what we see on the page. The body is often seperated into 'Header', 'Main' and 'Footer'. All of the elements are wrapped inside a 'HTML' element.

<!DOCTYPE html>
  <html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Site Title</title>
  </head>
  <body>
    <header>NavBar, title etc</header>
    <main>The main content of the page</main>
    <footer>Copyright text, useful links etc</footer>
  </body>
  </html>

Useful HTML Tags:

  • <p>Paragraph</p> - This is the main 'Text' block. As the name suggests, it's used for paragraphing.
  • <ul>Unordered List</ul> - A unordered list is used for 'Bullet Point lists.
  • <ol>Ordered List</ol> - A ordered list is used for sequencial "1. 2. 3." style lists.
  • <li>List Item</li> - As the name suggests, this tag wraps around any single item on your list.

CSS Styling

CSS, or Cascading Style Sheets, is the primary way of defining the look of a website. While it typically doesn't add new data, it converts the page from a black, white and blue page without any formatting to a lovely looking page.

As the name suggests, in CSS the styles 'cascade' downwards. A style set in the <body> element will cascade down to any tags inside it. This can be overridden if you redefine the style in a child object. In CSS, if you define a style twice, the one lowest in the style sheet will take priority.


<style>
body {
  color: green; -- This will change all text on the page to green. (other than anchor links...)
}
h1 {
  color: blue; -- This will override the h1 tag text to blue.
  color: pink; -- Since this is lower in the CSS, it will override the style and turn the h1 tag text to pink.
}
</style>

<body>
  <h1> Header </h1>
</body>
  • The !important rule will cause the style to override all other styles (unless the other style also has a !important rule, in which case specificity will take control.
  • Specificity causes styling to take priority in this order, from lowest priority to highest priority: tag, class, id, inline.
    • Tags have a specificity value of 1.
    • Classes have a specificity value of 10.
    • Ids have a specificity value of 100.
    • Inline have a specificity value of 1000.
  • The specificity of a certain rule is based on adding together the specificity of the selector. For example, if you have a selector with a tag, ID and Class, it would have the specificity of 111. For example: p.class#id{}. That's 1 + 10 + 100.

Inline Styling

When dealing with styles, with the exception of the !important rule, inline styles are always at the highest priority (a specificity value of 1000)

A inline style is a style defined within the tag that the style will be modifying. For example:

<p style="color:blue">Paragraph</p>

Flexbox

Flexbox is one of the two main layout features in CSS. The idea is to make a parent object a 'Flex Object'. Once the parent is set as a flex object it takes control of all it's children's positioning. Depending on the 'flex direction' of the parent, all the children will be lined up in a column or row. Flex makes it easy to dynamically allocate space to each child based on how many children there are and how much space there is to fill.

It's possible to tell a child to ignore the parent's instructions (Surprise surprise...). By combining the parent's flow and overriding a child's position, you can create lovely layouts. This is how you'd recreate the navigation bar on this page.


#navbar {
  display: flex;
  flex-direction: row;
  align-contents: flex-end;
}

#navbar header {
  align-self: flex-start;
}

Grid

Grid is the other main layout feature in CSS. While Flexbox is mostly used on one axis (Horizontal or vertical), grid, as you would expect from the name, works on the vertical and horizontal axis at the same time.

I personally prefer Grid over Flexbox for the simple reason of how much easier Grid works in regards to web responsiveness.

Grid has a 'grid-template-area' rule that allows you to define a 'grid' based on names. You can easily override this with media queries.


  main {
    display: grid;
    grid-template-area:
    "header header header"
    "sidebar main-content main-content"
    "footer footer footer";
  }

  @media screen and (max-width: 360px) {
    grid-template-area:
    "header header"
    "main-content main-content"
    "sidebar sidebar"
    "footer footer";
  }
